methods

A summary of the resources and process used to make these visualizations.

libraries and resources

All visualizations and data processing are done in javascript.

  • Jquery
  • Data Driven Documents, or d3.js, a web-based interactive visualization library
  • Colorbrewer2 by Cynthia Brewer

data processing

  1. First, human beings analyze a story, summarizing the main events, marking causal connections between events, and associating important characters, plot devices or tropes, and themes. In the case of Macbeth, the Genesis Project provided analysis of the story.
    Example of one individual's raw analysis of Sleeping Beauty
  2. Since the number of people who analyzed the stories was small, consensus on important events was reached by taking events, themes, tropes, and character sets that at least 3 out of the four coders agreed on.
  3. Data is encoded in the *story name here*_data.js files.
  4. All data processing and drawing methods are stored in *story name here*_methods.js files. Jquery and d3.js were the libraries used to generate all plots and interaction. Copies of all scripts are available on github.

alternative implementation libraries

Similar interactive visualizations can be generated by a variety of libraries, the best ones (ie the ones that would provide the easiest transition)

  • Java: Prefuse: An Information Visualization Toolkit
  • Actionscript: Prefuse: An Information Visualization Toolkit
  • Compatibility Note: d3.js, and therefore these interactive visualizations are accessible to most modern versions of browsers, but are not visible to older versions of Internet Explorer (IE 8,7, and 6) and as such they are best viewed in either Chrome or Firefox.

implementation steps for Java

Force diagrams in the prefuse library take in xml files detailing the links.

  1. Charts were implemented using a force diagram layout base. You can see an example/template layout for force layouts in the included demo set for prefuse.
  2. Anchor nodes along either the x or y axis, whichever you have chosen to denote chronology. In the case of these plots, I chose the x-axis from left to right for ascending time. An interesting idea for Java implementation would also be to include checkboxes for sorting via various criteria along the opposing axis, so one could bound the nodes in that category on the y axis if using our example as a template according to category.
  3. The prefuse wiki has many demo widgets that other people have added to force diagrams such as searching and highlighting for terms, which means that the java implementation has a realistic scope for showing much larger event sets that could be dynamically filtered or highlighted via search or category checkboxes, both of which are very easy to implement in prefuse (many examples exist that are essentially copy pasting in code).
